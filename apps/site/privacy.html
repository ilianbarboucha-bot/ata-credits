ATA Credits privacy summary

ATA Credits is designed so sponsor networks do not receive raw prompts, source code, file contents, repository names, API keys, or secret logs.

What sponsor providers may receive

  • placement context
  • tool name
  • route type
  • category label
  • approximate country
  • anonymous session identifier

What sponsor providers do not receive

  • raw prompt content
  • source code or pasted files
  • repository names
  • API keys, tokens, passwords, or secrets
  • raw AI response bodies by default

Operational notes

  • Credits are issued as pending first, then validated later.
  • Repeated or suspicious impression patterns can be rate-limited.
  • Official-mode requests are not proxied through the sponsored gateway.
